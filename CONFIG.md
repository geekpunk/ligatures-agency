# Album Configuration Guide

The album site is driven by a JSON configuration file located at `public/album-config.json`. This allows you to update all content without modifying the code.

## Configuration Structure

### Basic Information
```json
{
  "albumName": "Agency",
  "artistName": "Ligatures",
  "pageTitle": "Ligatures – Agency"
}
```

### Images
```json
{
  "images": {
    "front": "images/Agency-Front.png",
    "back": "images/Agency-Back.png"
  }
}
```
- Paths are relative to the `public/` directory
- Images should be in a standard web format (PNG, JPG)

### Colors
```json
{
  "colors": {
    "primary": "#5B96C7",
    "heroBackground": "#5B96C7",
    "background": "#1a1a1a",
    "text": "#e0e0e0"
  }
}
```
- **primary**: Main accent color used for links, buttons, track numbers, highlights (default: `#5B96C7`)
- **heroBackground**: Background color for the hero/header section (default: `#5B96C7`)
- **background**: Main page background color (default: `#1a1a1a`)
- **text**: Main text color (default: `#e0e0e0`)
- All colors must be valid CSS color values (hex, rgb, hsl, etc.)
- Colors are optional - if not specified, defaults will be used

### Tracks
```json
{
  "tracks": [
    {
      "side": "A",
      "number": "A1",
      "name": "All Kindsa Guys",
      "duration": "3'55\"",
      "file": "all_kindsa_guys.mp3",
      "lyrics": "Your lyrics here\nWith line breaks\nFor verses"
    }
  ]
}
```
- `side`: "A" or "B" for vinyl-style presentation
- `number`: Track identifier (e.g., "A1", "B2")
- `name`: Song title
- `duration`: Display duration (formatted string)
- `file`: MP3 filename in `public/songs/` directory
- `lyrics`: Song lyrics with `\n` for line breaks (optional)
  - Set to `null` or omit for instrumental tracks
  - If present and not null, a "Lyrics" button will appear in the player

### Credits
```json
{
  "credits": {
    "paragraphs": [
      "Simple text paragraph",
      {
        "text": "Text before link ",
        "link": {
          "text": "Link Text",
          "url": "https://example.com"
        },
        "suffix": " text after link."
      }
    ]
  }
}
```
- Can be simple strings or objects with embedded links
- Supports multiple links per paragraph (`link1`, `link2`)

### Social Links
```json
{
  "links": {
    "github": "https://github.com/...",
    "instagram": "https://instagram.com/...",
    "bandcamp": "https://bandcamp.com/..."
  }
}
```

### Other Settings
```json
{
  "footer": "Footer text here",
  "downloadZip": "ligatures-agency-album.zip"
}
```
- `footer`: Text displayed at bottom of page
- `downloadZip`: Filename of the album download ZIP in `public/`

## Updating Content

1. **Edit the config file**: `public/album-config.json`
2. **Update media files**: Add/update files in `public/images/` and `public/songs/`
3. **Rebuild**: Run `npm run build`
4. **Deploy**: Run `npm run deploy`

## File Organization

```
public/
├── album-config.json          # Main configuration
├── images/
│   ├── Agency-Front.png      # Album cover front
│   └── Agency-Back.png       # Album cover back
├── songs/
│   ├── track1.mp3
│   ├── track2.mp3
│   └── ...
└── album-name.zip            # Generated by build script
```

## Building the ZIP File

The build script automatically creates a ZIP file containing:
- All images from `images/`
- All songs from `public/songs/`

This runs automatically before each build via the `prebuild` script.

To manually create the ZIP:
```bash
npm run create-zip
```
